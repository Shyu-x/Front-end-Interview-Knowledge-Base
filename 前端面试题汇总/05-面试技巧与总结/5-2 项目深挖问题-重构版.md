## 2. WebEnv-OS 项目深挖（主项目）

### 2.1 一句话开场（30秒）

WebEnv 前端不是“单页堆功能”，而是 **桌面壳层 + 窗口编排内核 + 运行时接入** 三段式架构，目标是在浏览器中提供接近本地系统的 IDE 体验。

### 2.2 自顶向下架构拆解

#### A. 入口层

- `/desktop` 只承载桌面态
- IDE 业务与桌面交互解耦

#### B. 桌面壳层（主容器）

承担 5 类职责：

1. 布局策略：TopBar + Dock + Desktop Canvas
2. 应用注册：`availableApps` 元数据驱动
3. 窗口编排：统一 `create/close/focus/minimize/maximize/restore`
4. 视觉系统：壁纸与主题
5. 系统挂件：通知、监控、启动器

#### C. Desktop 画布

- 背景渲染：gradient/image/video
- 图标系统：选中态、点击态、定位
- 桌面上下文菜单：查看/刷新/新建

#### D. 窗口系统（核心）

- `WindowManager`：窗口集合、生命周期、`zIndex` 策略
- `Window`：拖拽、缩放、最大化/还原、最小尺寸约束

#### E. 系统级交互

- TopBar：菜单 + 系统状态
- Dock：启动、活跃态、窗口数、右键关闭全部

#### F. 启动与错误兜底

- `useKernel`：启动流程与进度
- `KernelProvider`：加载/错误/完成态统一渲染
- `crossOriginIsolated` 不满足时提供诊断页

#### G. 状态分层

1. 编辑域
2. 界面域
3. 工程域
4. 环境域
5. 偏好域

### 2.3 高频追问（标准答法）

1. 桌面怎么实现：壳层 + 窗口编排 + 原子窗口
2. 为什么不是 UI 拼装：有系统语义和生命周期约束
3. 最难点：交互拟真 + 运行时约束

## 3. FastDocument 项目深挖（协作主项目）

### 3.1 一句话开场（30秒）

FastDocument 是 **块级编辑模型 + 实时协作状态机 + 协作感知 UI** 的在线文档系统，核心是协作一致性与可恢复性。

### 3.2 自顶向下架构拆解

1. 应用壳层：Sidebar + Header + 主区场景切换
2. 信息架构：导航 + 协作工具 + 工作区树
3. 编辑器核心：`Block[]`、按 `type` 渲染、Slash/Markdown 输入
4. 协作状态机：本地动作/远端动作/文档动作分离
5. 协作 UI：SocketClient + ChatDrawer + Outline + MenuBar
6. 音视频：LiveKit 动态加载 + 轨道事件绑定

### 3.3 高频追问（标准答法）

1. 为什么块模型：冲突域更小，局部更新更稳
2. 一致性策略：最终一致，低延迟优先
3. OT vs CRDT：协作编辑优先考虑 CRDT（如 Y.js）

## 6. 安全与性能（原文重点归并）

### 6.1 安全

- XSS：过滤 + 编码 + CSP
- CSRF：Token + SameSite + Referer/Origin
- 点击劫持：X-Frame-Options + frame-ancestors

### 6.2 性能模型

$$
T_{ui} = T_{event} + T_{state} + T_{render} + T_{io}
$$

### 6.3 优化清单

1. 首屏：分包、懒加载、SSR/SSG、预加载
2. 渲染：减少重排重绘、虚拟列表、批量 DOM
3. 内存：清理监听/定时器/订阅、排查闭包泄漏
4. 图片：WebP/AVIF、`srcset`、懒加载、占位图
5. 接口：合并、去重、防抖、重试、并发编排
6. 缓存：内存 + HTTP + SW + 持久化

## 9. 面试技巧与高压追问

### 9.1 STAR

- S 情境
- T 任务
- A 行动
- R 结果

### 9.2 三档回答

- 30 秒：定位 + 核心点 + 结果
- 90 秒：链路 + 难点 + 结果
- 5 分钟：原理 + 取舍 + 演进

### 9.3 高频追问防守句式

1. “不是强一致会丢数据？” -> 最终一致 + 补偿/重试/重连快照
2. “为什么不用更重架构？” -> 阶段性权衡，阈值后演进
3. “怎么证明不是拍脑袋？” -> 约束驱动 + 方案对比 + 指标验证
