## 1. React 基础概念

### 1.1 什么是React？它有什么特点？

**问题**：请介绍一下React，以及它有哪些核心特点？

**答案**：

React 是 Facebook 于 2013 年开源的一个用于构建用户界面的 JavaScript 库，其核心特点如下：

#### 核心特点

1. **声明式编程（Declarative）**
   - React 采用声明式编程范式，开发者只需要描述 UI 应该是什么状态，React 会自动处理 DOM 更新
   - 代码更加可预测，易于调试

2. **组件化（Component-Based）**
   - React 应用由独立、可复用的组件构成
   - 每个组件维护自己的状态和逻辑
   - 提高代码复用性和可维护性

3. **虚拟DOM（Virtual DOM）**
   - React 在内存中维护一个虚拟 DOM 树
   - 当状态变化时，通过 Diff 算法比较新旧虚拟 DOM
   - 只更新实际变化的 DOM 节点，提高性能

4. **单向数据流（Unidirectional Data Flow）**
   - 数据沿着组件树从上向下流动
   - 状态变化通过 setState 触发重新渲染
   - 使应用行为更容易预测

5. **JSX 语法扩展**
   - JSX 是 JavaScript 的语法扩展，允许在 JavaScript 中编写 HTML -like 代码
   - 最终会被编译为普通的 JavaScript 函数调用

#### 实际代码示例

```jsx
// 声明式的组件写法
function Welcome({ name }) {
  return <h1>Hello, {name}!</h1>;
}

// 组件化示例
function App() {
  return (
    <div>
      <Welcome name="World" />
      <Welcome name="React" />
    </div>
  );
}
```

#### 教学提示

> React 是一个**视图层**库，而不是完整的 MVC/MVVM 框架。它专注于 UI 层面的开发，状态管理和路由等功能需要配合其他库使用。理解这一点有助于你在实际项目中合理选型。

### 1.2 React与其他前端框架相比有什么优势和劣势？

**问题**：React 与 Vue、Angular 相比，有什么优势和劣势？

**答案**：

#### 三大框架对比

| 特性 | React | Vue | Angular |
|------|-------|-----|---------|
| 创始人 | Facebook |尤雨溪 | Google |
| 首次发布 | 2013年 | 2014年 | 2010年 (AngularJS), 2016年 (Angular 2+) |
| 学习曲线 | 中等 | 较平缓 | 较陡峭 |
| 生态成熟度 | 非常成熟 | 成熟 | 成熟 |
| 社区活跃度 | 非常活跃 | 活跃 | 活跃 |
| 核心库大小 | ~40KB | ~60KB | ~500KB |

#### React 的优势

1. **灵活的生态**
   - React 只是一个视图层库，你可以自由选择其他库组合
   - 路由（React Router）、状态管理（Redux/Zustand）等都是可选的

2. **强大的社区支持**
   - 全球最流行的前端框架之一
   - 丰富的第三方库和工具

3. **JSX 的灵活性**
   - JSX 将模板与逻辑结合在一起
   - 可以使用 JavaScript 的全部能力来处理模板

4. **Fiber 架构**
   - React 16 引入的 Fiber 架构支持异步渲染
   - 为未来性能优化奠定基础

5. **React Native**
   - 使用 React 语法开发原生移动应用
   - 一套代码同时支持 iOS 和 Android

#### React 的劣势

1. **需要配合其他库**
   - 路由、状态管理需要额外学习
   - 新手可能不知道如何选择

2. **jsx 语法学习曲线**
   - 对于不熟悉 JS 的人来说，JSX 可能需要适应

3. **文档不如 Vue 友好**
   - React 文档相对分散，学习路径不如 Vue 清晰

#### 教学提示

> 选择框架时，要根据项目需求、团队技术栈和社区支持来综合考虑。没有最好的框架，只有最适合的框架。React 的灵活性是其最大优势，也是其最大的挑战——你需要做更多的技术决策。

## 2. 组件与Props

### 2.1 函数组件与类组件的区别？

**问题**：React 中的函数组件和类组件有什么区别？应该使用哪一个？

**答案**：

#### 基本区别

```jsx
// 函数组件（Function Component）
function Welcome(props) {
  return <h1>Hello, {props.name}!</h1>;
}

// 类组件（Class Component）
class Welcome extends React.Component {
  render() {
    return <h1>Hello, {this.props.name}!</h1>;
  }
}
```

#### 核心区别对比

| 特性 | 函数组件 | 类组件 |
|------|----------|--------|
| 语法 | 函数 | ES6 类 |
| this | 无需处理 | 需要 bind 或箭头函数 |
| 状态 | Hooks (useState) | this.state |
| 生命周期 | Hooks (useEffect) | 生命周期方法 |
| 性能 | 较轻量 | 较重（需要继承） |
| 可读性 | 简洁清晰 | 相对冗长 |
| 推荐 | **首选** | 不推荐新代码使用 |

#### 为什么优先使用函数组件？

1. **代码更简洁**
   - 函数组件通常只需要 10-20 行代码
   - 类组件可能需要 50+ 行

2. **更好的性能**
   - 函数组件不需要实例化类
   - 生产环境测试显示函数组件性能略优

3. **Hooks 的强大能力**
   - useState、useEffect 等 Hooks 提供了比类组件生命周期更灵活的能力
   - 自定义 Hooks 实现了逻辑复用

4. **更容易测试**
   - 函数是更天然的测试单元
   - 不需要模拟 React 组件实例

#### 实际对比示例

```jsx
// 类组件写法
class Counter extends React.Component {
  constructor(props) {
    super(props);
    this.state = { count: 0 };
    this.handleClick = this.handleClick.bind(this);
  }

  handleClick() {
    this.setState(prev => ({ count: prev.count + 1 }));
  }

  render() {
    return (
      <div>
        <p>Count: {this.state.count}</p>
        <button onClick={this.handleClick}>Increment</button>
      </div>
    );
  }
}

// 函数组件 + Hooks 写法
function Counter() {
  const [count, setCount] = useState(0);

  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={() => setCount(c => c + 1)}>Increment</button>
    </div>
  );
}
```

#### 教学提示

> 虽然 React 仍然支持类组件，但官方明确表示未来会继续支持，但新代码应优先使用函数组件。类组件的 props、state、生命周期方法等概念在 Hooks 中都有对应的实现方式。

### 2.2 什么是Props？Props有哪些特性？

**问题**：请详细解释 React 中的 Props 是什么，以及它的特性。

**答案**：

#### Props 的定义

Props（Properties）是 React 组件之间传递数据的主要方式。Props 是只读的，父组件通过 Props 向子组件传递数据。

#### Props 的特性

1. **只读性**
   - 子组件不能修改 props
   - Props 是单向数据流

2. **可传递任意数据**
   - 可以传递字符串、数字、布尔值
   - 可以传递数组、对象
   - 可以传递函数
   - 可以传递 React 元素

3. **默认值设置**
   - 可以为 props 设置默认值

4. **类型检查**
   - 可以使用 PropTypes 或 TypeScript 进行类型检查

#### 代码示例

```jsx
// 父组件
function Parent() {
  const user = { name: 'Alice', age: 25 };
  const handleAction = () => console.log('Action!');

  return (
    <Child
      name="Bob"           // 字符串
      count={42}           // 数字
      isActive={true}      // 布尔值
      items={[1, 2, 3]}    // 数组
      user={user}          // 对象
      onAction={handleAction} // 函数
      header={<h1>Title</h1>} // React 元素
    />
  );
}

// 子组件
function Child(props) {
  return (
    <div>
      <p>Name: {props.name}</p>
      <p>Count: {props.count}</p>
      <p>Active: {props.isActive ? 'Yes' : 'No'}</p>
      <p>Items: {props.items.join(', ')}</p>
      <p>User: {props.user.name}, {props.user.age}</p>
      <button onClick={props.onAction}>Click</button>
      {props.header}
    </div>
  );
}
```

#### Props 默认值

```jsx
// 使用 defaultProps（传统方式）
function Button({ text = 'Click me', onClick }) {
  return <button onClick={onClick}>{text}</button>;
}
Button.defaultProps = {
  text: 'Click me'
};
```

#### Props 类型检查

```jsx
import PropTypes from 'prop-types';

function UserCard({ name, age, email, isAdmin }) {
  return (
    <div>
      <h2>{name}</h2>
      <p>Age: {age}</p>
      <p>Email: {email}</p>
      <p>Admin: {isAdmin ? 'Yes' : 'No'}</p>
    </div>
  );
}

UserCard.propTypes = {
  name: PropTypes.string.isRequired,
  age: PropTypes.number,
  email: PropTypes.string.isRequired,
  isAdmin: PropTypes.bool
};
```

#### 教学提示

> Props 是 React 数据流的基础。记住：**Props 是只读的**——这是 React 最重要的原则之一。如果需要子组件修改数据，应该使用状态提升（State Lifting），将状态提升到父组件中，通过回调函数来修改。

### 2.3 什么是组件组合（Component Composition）？

**问题**：请解释 React 中的组件组合是什么，如何使用？

**答案**：

#### 组件组合的定义

组件组合（Component Composition）是 React 中一种重要的设计模式，通过将组件嵌套组合来构建复杂的 UI。它利用 props.children 或 props 传递组件来实现。

#### 组合模式

1. **使用 children prop**

```jsx
// 容器组件
function Card({ children, title }) {
  return (
    <div className="card">
      <div className="card-header">
        <h2>{title}</h2>
      </div>
      <div className="card-body">
        {children}
      </div>
    </div>
  );
}

// 使用
function App() {
  return (
    <Card title="Welcome">
      <p>This is the card content.</p>
      <button>Learn More</button>
    </Card>
  );
}
```

2. **使用 Render Props**

```jsx
// 父组件提供数据和渲染函数
function MouseTracker({ render }) {
  const [position, setPosition] = useState({ x: 0, y: 0 });

  useEffect(() => {
    const handleMouseMove = (e) => {
      setPosition({ x: e.clientX, y: e.clientY });
    };
    window.addEventListener('mousemove', handleMouseMove);
    return () => window.removeEventListener('mousemove', handleMouseMove);
  }, []);

  return render(position);
}

// 使用
function App() {
  return (
    <MouseTracker
      render={({ x, y }) => (
        <h1>Mouse position: {x}, {y}</h1>
      )}
    />
  );
}
```

3. **使用高阶组件（HOC）**

```jsx
// 高阶组件：添加日志功能
function withLogger(WrappedComponent) {
  return function(props) {
    console.log(`Rendering ${WrappedComponent.name}`);
    return <WrappedComponent {...props} />;
  };
}

// 使用
const EnhancedComponent = withLogger(MyComponent);
```

4. **使用 Props 传递组件**

```jsx
// 布局组件
function Layout({ header, main, footer }) {
  return (
    <div>
      <header>{header}</header>
      <main>{main}</main>
      <footer>{footer}</footer>
    </div>
  );
}

// 使用
function App() {
  return (
    <Layout
      header={<Nav />}
      main={<Content />}
      footer={<Footer />}
    />
  );
}
```

#### 组合的优势

1. **提高复用性**
   - 容器组件可以包装不同内容

2. **关注点分离**
   - 逻辑和 UI 分离

3. **灵活的结构**
   - 组件可以自由组合

#### 教学提示

> 组件组合是 React 的核心模式之一。掌握这种模式可以帮助你写出更灵活、可复用的组件。React 的哲学是"组合优于继承"，通过组合可以构建任何复杂的 UI，而不需要使用传统的继承模式。

## 3. State 与生命周期

### 3.1 什么是State？它与Props有什么区别？

**问题**：请解释 React 中的 State 是什么，它与 Props 有什么区别？

**答案**：

#### State 的定义

State 是组件内部管理的状态数据，用于存储组件的私有数据。当 state 发生变化时，组件会重新渲染。

#### State 与 Props 的区别

| 特性 | State | Props |
|------|-------|-------|
| 定义位置 | 组件内部 | 组件外部传入 |
| 修改方式 | 使用 setState | 父组件传递 |
| 职责 | 组件内部数据管理 | 组件间数据传递 |
| 响应变化 | 会触发重新渲染 | 变化会触发重新渲染 |
| 初始值 | 可设置初始值 | 从父组件获取 |
| 跨组件共享 | 不能直接共享 | 可以传递 |

#### State 使用示例

```jsx
function Counter() {
  // 声明状态
  const [count, setCount] = useState(0);
  const [user, setUser] = useState({ name: 'Alice', age: 25 });

  // 更新状态
  const increment = () => {
    setCount(prevCount => prevCount + 1);
  };

  // 更新对象状态
  const updateUser = () => {
    setUser(prevUser => ({
      ...prevUser,
      age: prevUser.age + 1
    }));
  };

  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={increment}>Increment</button>
      <p>User: {user.name}, {user.age}</p>
      <button onClick={updateUser}>Update Age</button>
    </div>
  );
}
```

#### 什么时候使用 State？

1. **组件内部需要追踪的数据**
   - 计数器当前值
   - 表单输入值
   - 展开/折叠状态

2. **不需要共享给其他组件的数据**

3. **会随时间或交互变化的数据**

#### 什么时候使用 Props？

1. **从父组件传递的数据**
2. **回调函数**
3. **组件配置**
4. **需要共享的状态**

#### 教学提示

> 理解 State 和 Props 的区别是 React 开发的基础。记住：**State 是组件的"私有财产"，Props 是组件的"输入参数"**。合理地划分 State 和 Props 是构建良好 React 应用的关键。

### 3.2 setState是同步还是异步的？

**问题**：React 中的 setState 是同步还是异步的？它在什么时候会异步更新？

**答案**：

#### setState 的更新机制

在 React 18 中，setState 的行为取决于执行上下文：

1. **在 React 事件处理中（异步）**
   - 在 onClick、onChange 等事件处理函数中
   - React 会批量更新（Batching）
   - 不会立即反映到 DOM 上

2. **在原生事件、setTimeout、Promise 中（同步）**
   - 在 addEventListener、setTimeout、Promise.then 中
   - 会立即更新

#### 代码示例

```jsx
function Counter() {
  const [count, setCount] = useState(0);

  // 事件处理中的批量更新
  const handleClick = () => {
    // 这里的 setState 会被批量处理
    setCount(c => c + 1); // 计划: 0 -> 1
    setCount(c => c + 1); // 计划: 1 -> 2
    setCount(c => c + 1); // 计划: 2 -> 3
    // 最终 count = 3，只触发一次重渲染
  };

  // setTimeout 中的同步更新
  useEffect(() => {
    const timer = setTimeout(() => {
      setCount(c => c + 1); // 立即执行
      setCount(c => c + 1); // 立即执行
    }, 1000);
    return () => clearTimeout(timer);
  }, []);

  return <div>Count: {count}</div>;
}
```

#### 类组件中的 setState

```jsx
class Counter extends React.Component {
  state = { count: 0 };

  handleClick = () => {
    // 批量更新
    this.setState({ count: this.state.count + 1 });
    this.setState({ count: this.state.count + 1 });
    // 不会立即生效，count 仍然显示 1
    console.log(this.state.count); // 0
  };

  handleClickSync = () => {
    // 使用回调函数获取最新值
    this.setState(
      (prevState) => ({ count: prevState.count + 1 }),
      () => console.log(this.state.count) // 回调中能获取最新值
    );
  };
}
```

#### 函数组件中的优化

```jsx
function Counter() {
  const [count, setCount] = useState(0);

  // 多次更新优化
  const handleClick = () => {
    // 使用函数式更新，React 会自动合并更新
    setCount(prev => prev + 1);
    setCount(prev => prev + 1);
    setCount(prev => prev + 1);
    // 最终只触发一次重渲染，count = 3
  };

  // 依赖多个状态时的优化
  const [state, setState] = useState({ count: 0, name: 'Alice' });

  const updateState = () => {
    setState(prev => ({ ...prev, count: prev.count + 1 }));
    // 不会丢失其他状态
  };

  return <div>Count: {count}</div>;
}
```

#### React 18 的自动批处理

```jsx
// React 18 中，所有 setState 都会自动批处理
function handleClick() {
  setCount(c => c + 1);
  setName('Bob');
  // 无论在哪里调用，都会批量更新
}
```

#### 教学提示

> 理解 setState 的异步机制对于正确使用 React 至关重要。记住：**不要依赖 this.state 来计算下一个状态**，而应该使用函数式更新 `setState(prevState => ...)`。这不仅能确保获取到最新值，还能享受 React 的批处理优化。

## 4. Hooks 详解

### 4.1 useState 的工作原理？

**问题**：请详细解释 useState Hook 的工作原理和使用方法。

**答案**：

#### useState 基础

```jsx
const [state, setState] = useState(initialValue);
```

- **state**: 当前状态值
- **setState**: 更新状态的函数
- **initialValue**: 状态的初始值

#### 基本用法

```jsx
function Counter() {
  const [count, setCount] = useState(0);

  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={() => setCount(count + 1)}>Add</button>
    </div>
  );
}
```

#### useState 的特性

1. **函数式更新**
   - 当新状态依赖旧状态时，使用函数避免闭包问题

```jsx
// 错误写法
const [count, setCount] = useState(0);
const increment = () => setCount(count + 1); // 闭包陷阱

// 正确写法
const increment = () => setCount(prevCount => prevCount + 1);
```

2. **惰性初始化**
   - 复杂计算的状态，使用函数延迟初始化

```jsx
// 每次渲染都会执行
const [state, setState] = useState(expensiveCompute());

// 只在首次渲染时执行
const [state, setState] = useState(() => expensiveCompute());
```

3. **多个状态**
   - 使用多个 useState 管理不同状态

```jsx
function Form() {
  const [name, setName] = useState('');
  const [email, setEmail] = useState('');
  const [age, setAge] = useState(0);

  return (
    <form>
      <input value={name} onChange={e => setName(e.target.value)} />
      <input value={email} onChange={e => setEmail(e.target.value)} />
      <input value={age} onChange={e => setAge(e.target.value)} />
    </form>
  );
}
```

4. **对象状态**
   - 需要使用展开运算符合并状态

```jsx
function Form() {
  const [formData, setFormData] = useState({
    name: '',
    email: '',
    age: 0
  });

  const updateField = (field, value) => {
    setFormData(prev => ({
      ...prev,
      [field]: value
    }));
  };

  return (
    <form>
      <input
        value={formData.name}
        onChange={e => updateField('name', e.target.value)}
      />
    </form>
  );
}
```

#### 模拟 useState 实现

```jsx
// 简化的 useState 实现
let currentComponent = null;
let hookIndex = 0;
const stateList = [];

function useState(initialValue) {
  const currentIndex = hookIndex;

  // 首次渲染时初始化状态
  if (stateList.length === currentIndex) {
    stateList.push(initialValue);
  }

  // 存储当前索引的引用，供 setState 使用
  const setState = (newValue) => {
    // 支持函数式更新
    const value = typeof newValue === 'function'
      ? newValue(stateList[currentIndex])
      : newValue;

    if (value !== stateList[currentIndex]) {
      stateList[currentIndex] = value;
      // 触发重新渲染
      render(currentComponent);
    }
  };

  hookIndex++;
  return [stateList[currentIndex], setState];
}
```

#### 教学提示

> useState 是 React Hooks 系统的基础。记住几个关键点：
> 1. 使用函数式更新来处理依赖前一状态的情况
> 2. 复杂计算使用惰性初始化避免性能问题
> 3. 对象状态需要手动合并（使用展开运算符）
> 4. useState 在组件顶层调用，不要在循环、条件判断中使用

### 4.2 useEffect 的执行时机和依赖数组？

**问题**：请详细解释 useEffect 的执行时机，依赖数组的作用，以及各种使用场景。

**答案**：

#### useEffect 基础

```jsx
useEffect(effectFunction, dependencyArray);
```

- **effectFunction**: 副作用函数
- **dependencyArray**: 依赖数组

#### 执行时机

1. **组件挂载后**（componentDidMount）
   - 首次渲染后执行

2. **组件更新后**（componentDidUpdate）
   - 依赖变化时执行

3. **组件卸载前**（componentWillUnmount）
   - 返回清理函数时执行

#### 依赖数组的不同用法

```jsx
function MyComponent() {
  const [count, setCount] = useState(0);
  const [data, setData] = useState(null);

  // 1. 不写依赖数组：每次渲染都执行
  useEffect(() => {
    console.log('Every render');
  });

  // 2. 空依赖数组：只执行一次（挂载时）
  useEffect(() => {
    console.log('Mount only');
    // 相当于 componentDidMount
  }, []);

  // 3. 有依赖：依赖变化时执行
  useEffect(() => {
    console.log('Count changed:', count);
  }, [count]);

  // 4. 多个依赖
  useEffect(() => {
    console.log('Count or data changed');
  }, [count, data]);

  return <div>Count: {count}</div>;
}
```

#### 清理函数

```jsx
function MouseTracker() {
  const [position, setPosition] = useState({ x: 0, y: 0 });

  useEffect(() => {
    // 1. 设置事件监听
    const handleMouseMove = (event) => {
      setPosition({ x: event.clientX, y: event.clientY });
    };

    window.addEventListener('mousemove', handleMouseMove);

    // 2. 返回清理函数（组件卸载时执行）
    return () => {
      window.removeEventListener('mousemove', handleMouseMove);
    };
  }, []); // 空依赖，只执行一次

  return <div>Position: {position.x}, {position.y}</div>;
}
```

#### 常见使用场景

1. **数据获取（Data Fetching）**

```jsx
function UserProfile({ userId }) {
  const [user, setUser] = useState(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);

  useEffect(() => {
    setLoading(true);
    setError(null);

    fetch(`/api/users/${userId}`)
      .then(res => res.json())
      .then(data => {
        setUser(data);
        setLoading(false);
      })
      .catch(err => {
        setError(err.message);
        setLoading(false);
      });
  }, [userId]);

  if (loading) return <div>Loading...</div>;
  if (error) return <div>Error: {error}</div>;
  return <div>{user.name}</div>;
}
```

2. **订阅外部数据源**

```jsx
function ChatRoom({ roomId }) {
  const [messages, setMessages] = useState([]);

  useEffect(() => {
    const subscription = chat.subscribe(roomId, (newMessage) => {
      setMessages(prev => [...prev, newMessage]);
    });

    // 清理订阅
    return () => subscription.unsubscribe();
  }, [roomId]);

  return <div>{messages.map(m => <Message key={m.id} text={m.text} />)}</div>;
}
```

3. **操作 DOM**

```jsx
function AutoFocus() {
  const inputRef = useRef(null);

  useEffect(() => {
    inputRef.current.focus();
  }, []);

  return <input ref={inputRef} />;
}
```

4. **定时器**

```jsx
function Timer() {
  const [seconds, setSeconds] = useState(0);

  useEffect(() => {
    const interval = setInterval(() => {
      setSeconds(s => s + 1);
    }, 1000);

    return () => clearInterval(interval);
  }, []);

  return <div>Seconds: {seconds}</div>;
}
```

#### 常见错误

```jsx
function Counter() {
  const [count, setCount] = useState(0);

  // 错误1：依赖数组为空，但使用了 count
  useEffect(() => {
    console.log(count); // 警告：count 应该在依赖数组中
  }, []);

  // 错误2：使用 setState 导致无限循环
  useEffect(() => {
    if (count < 10) {
      setCount(count + 1); // 会导致无限循环
    }
  }, [count]);

  // 正确：使用函数式更新
  useEffect(() => {
    if (count < 10) {
      setTimeout(() => setCount(c => c + 1), 100);
    }
  }, [count]); // 依赖 count，但不会导致无限循环

  return <div>{count}</div>;
}
```

#### 教学提示

> useEffect 是 React 中处理副作用的主要方式。记住：
> 1. 依赖数组决定了 effect 何时执行
> 2. 清理函数用于防止内存泄漏
> 3. 使用函数式更新避免闭包陷阱
> 4. ESLint 的 exhaustive-deps 规则能帮助检查依赖是否正确
> 5. 理解"依赖变化时重新执行"的逻辑是正确使用 useEffect 的关键

### 4.3 useRef 的使用场景？

**问题**：useRef 在 React 中有哪些使用场景？请举例说明。

**答案**：

#### useRef 基础

```jsx
const refContainer = useRef(initialValue);
```

- `useRef` 返回一个可变的 ref 对象
- `ref.current` 属性可以存储任意值
- 改变 `ref.current` 不会触发重新渲染

#### 主要使用场景

1. **访问 DOM 元素**

```jsx
function TextInput() {
  const inputRef = useRef(null);
  const focusInput = () => {
    inputRef.current.focus();
  };

  return (
    <div>
      <input ref={inputRef} type="text" />
      <button onClick={focusInput}>Focus Input</button>
    </div>
  );
}

// 多个 ref
function MultiRefs() {
  const refs = useRef([]);

  const handleClick = (index) => {
    refs.current[index].focus();
  };

  return (
    <div>
      {[0, 1, 2].map(i => (
        <input key={i} ref={el => refs.current[i] = el} />
      ))}
      <button onClick={() => handleClick(1)}>Focus Second</button>
    </div>
  );
}
```

2. **存储不需要触发渲染的值**

```jsx
function Timer() {
  const [count, setCount] = useState(0);
  const countRef = useRef(0); // 存储真实计数
  const timerRef = useRef(null);

  const startTimer = () => {
    timerRef.current = setInterval(() => {
      countRef.current += 1;
      console.log('Count:', countRef.current);
    }, 1000);
  };

  const stopTimer = () => {
    clearInterval(timerRef.current);
  };

  // 显示的 count 不会自动更新
  return (
    <div>
      <p>Display: {count}</p>
      <button onClick={startTimer}>Start</button>
      <button onClick={stopTimer}>Stop</button>
    </div>
  );
}
```

3. **在 useEffect 中访问最新值**

```jsx
function MyComponent() {
  const [value, setValue] = useState('');

  useEffect(() => {
    // 如果直接使用 value，会是闭包中的旧值
    // 使用 ref 可以获取最新值
    const subscription = someAPI.subscribe(valueRef.current);
    return () => subscription.unsubscribe();
  }, []); // 空依赖，但能访问最新值

  const valueRef = useRef(value);
  valueRef.current = value; // 每次渲染时更新 ref

  return <input value={value} onChange={e => setValue(e.target.value)} />;
}
```

4. **实现高性能计数器**

```jsx
function HighPerfCounter() {
  const [count, setCount] = useState(0);
  const countRef = useRef(0);

  const increment = () => {
    countRef.current += 1;
    // 使用 ref 更新，不触发重新渲染
  };

  const display = () => {
    setCount(countRef.current); // 需要显示时才触发渲染
  };

  return (
    <div>
      <button onClick={increment}>Increment (no render)</button>
      <button onClick={display}>Display</button>
      <p>Count: {count}</p>
    </div>
  );
}
```

5. **保存上一次的 Props 或 State**

```jsx
function MyComponent({ value }) {
  const prevValueRef = useRef();

  useEffect(() => {
    prevValueRef.current = value;
  }, [value]);

  const prevValue = prevValueRef.current;

  return (
    <div>
      <p>Current: {value}</p>
      <p>Previous: {prevValue}</p>
    </div>
  );
}
```

#### useRef vs useState

| 特性 | useRef | useState |
|------|--------|----------|
| 更新后触发重新渲染 | 否 | 是 |
| 初始值 | 仅首次使用 | 每次渲染使用 |
| 返回值 | { current: value } | [value, setValue] |
| 异步更新 | 同步 | 异步（批处理） |

#### 教学提示

> useRef 是 React 中一个"特殊"的 Hook：
> 1. 它是最不像 Hook 的 Hook——改变它不会触发重新渲染
> 2. 它主要用于：访问 DOM、存储跨渲染的数据、不需要触发渲染的值
> 3. useRef.current 就像一个"实例变量"——它是可变的且不与渲染绑定
> 4. 合理使用 useRef 可以帮助你避免闭包陷阱和性能问题

### 4.4 useMemo 和 useCallback 的区别？

**问题**：useMemo 和 useCallback 有什么区别？应该在什么时候使用它们？

**答案**：

#### 基本定义

```jsx
// 缓存计算结果
const memoizedValue = useMemo(() => computeExpensiveValue(a, b), [a, b]);

// 缓存函数
const memoizedCallback = useCallback(() => doSomething(a, b), [a, b]);
```

#### 区别对比

| 特性 | useMemo | useCallback |
|------|---------|-------------|
| 作用 | 缓存计算结果 | 缓存函数 |
| 返回值 | 任意值 | 函数 |
| 本质 | `useMemo(fn, deps)` = `useCallback(fn(), deps)` |

#### useMemo 使用场景

```jsx
function ExpensiveComponent({ list, filter }) {
  // 复杂的过滤和排序操作
  const filteredList = useMemo(() => {
    console.log('Computing filtered list...');
    return list
      .filter(item => item.name.includes(filter))
      .sort((a, b) => a.name.localeCompare(b.name));
  }, [list, filter]);

  return (
    <ul>
      {filteredList.map(item => (
        <li key={item.id}>{item.name}</li>
      ))}
    </ul>
  );
}
```

#### useCallback 使用场景

```jsx
function Parent() {
  const [count, setCount] = useState(0);

  // 每次渲染都会创建新函数
  const handleClick = () => {
    console.log('clicked');
  };

  // 使用 useCallback 缓存函数
  const handleClickMemoized = useCallback(() => {
    console.log('clicked');
  }, []); // 空依赖，函数永远不会变

  // 依赖变化的回调
  const handleAdd = useCallback((amount) => {
    setCount(c => c + amount);
  }, []); // setCount 永远是稳定的

  return (
    <div>
      <Child onClick={handleClickMemoized} onAdd={handleAdd} />
    </div>
  );
}

function Child({ onClick, onAdd }) {
  // Child 只会在 onClick 或 onAdd 变化时重新渲染
  return (
    <div>
      <button onClick={onClick}>Click</button>
      <button onClick={() => onAdd(1)}>Add 1</button>
    </div>
  );
}
```

#### 何时使用？

**useMemo 的使用场景**：
1. 昂贵的计算
2. 引用相等性重要时（传递给子组件的 object/array）
3. 复杂的数据转换

```jsx
// 场景1：昂贵计算
const sortedData = useMemo(() => {
  return data.sort((a, b) => b.value - a.value);
}, [data]);

// 场景2：引用相等（避免子组件不必要渲染）
const options = useMemo(() => ({
  theme: 'dark',
  language: 'en'
}), []); // 每次渲染都是同一个对象

<Select options={options} />
```

**useCallback 的使用场景**：
1. 传递给子组件的回调函数
2. 依赖变化的回调（在 useEffect 中使用）
3. 事件处理函数

```jsx
// 场景1：传递给 memoized 子组件
const handleSubmit = useCallback((data) => {
  console.log(data);
}, []);

<ExpensiveChild onSubmit={handleSubmit} />

// 场景2：在 useEffect 中使用
useEffect(() => {
  const handler = (e) => console.log(e);
  window.addEventListener('resize', handler);
  return () => window.removeEventListener('resize', handler);
}, [handleCallback]);
```

#### 不要过度使用

```jsx
function MyComponent() {
  const [count, setCount] = useState(0);

  // 不必要的 useMemo
  // 简单计算不需要 memoization
  const doubled = useMemo(() => count * 2, [count]);

  // 不必要的 useCallback
  // 只有当函数传递给 memoized 子组件时才有意义
  const handleClick = useCallback(() => {
    setCount(c => c + 1);
  }, []);

  return <button onClick={handleClick}>{doubled}</button>;
}
```

#### 教学提示

> useMemo 和 useCallback 都是用于性能优化的 Hook，但：
> 1. 不要过早优化——先写正确代码，再根据性能分析决定是否使用
> 2. 过度使用会增加代码复杂度，可能弊大于利
> 3. React DevTools 的 Profiler 可以帮助你识别性能瓶颈
> 4. 记住：只有当子组件使用了 React.memo，且你传递的是对象或函数时，useMemo/useCallback 才有意义

## 5. 虚拟DOM与Diff算法

### 5.1 什么是虚拟DOM？

**问题**：请解释 React 中的虚拟 DOM 是什么，以及它的工作原理。

**答案**：

#### 虚拟 DOM 定义

虚拟 DOM（Virtual DOM）是 React 内部使用的一种 JavaScript 对象结构，它是对真实 DOM 的 JavaScript 映射表示。

#### 工作原理

```
用户操作 → 状态更新 → 虚拟 DOM 更新 → Diff 算法对比 → 最小化真实 DOM 操作 → 界面更新
```

#### 代码示例

```jsx
// React 组件
function Hello() {
  return <h1>Hello, World!</h1>;
}

// 虚拟 DOM 结构（简化版）
{
  type: 'h1',
  props: {
    children: 'Hello, World!'
  }
}

// 复杂结构
function App() {
  return (
    <div className="container">
      <h1>Title</h1>
      <ul>
        <li>Item 1</li>
        <li>Item 2</li>
      </ul>
    </div>
  );
}

// 对应的虚拟 DOM
{
  type: 'div',
  props: { className: 'container' },
  children: [
    { type: 'h1', props: { children: 'Title' } },
    {
      type: 'ul',
      props: {},
      children: [
        { type: 'li', props: { children: 'Item 1' } },
        { type: 'li', props: { children: 'Item 2' } }
      ]
    }
  ]
}
```

#### 虚拟 DOM 的优势

1. **性能优化**
   - 减少直接操作真实 DOM
   - 批量更新，最小化重排重绘

2. **跨平台支持**
   - React Native：虚拟 DOM → 原生组件
   - React VR：虚拟 DOM → 3D 场景

3. **声明式 UI**
   - 只需描述 UI 状态，React 处理 DOM 更新

4. **开发体验**
   - JSX 使代码更易读和维护
   - 错误提示更友好

#### 虚拟 DOM 的劣势

1. **首次渲染开销**
   - 需要创建完整的虚拟 DOM 树

2. **内存占用**
   - JavaScript 对象占用内存

3. **不一定最快**
   - 小型简单界面可能直接操作 DOM 更快

#### 教学提示

> 虚拟 DOM 是 React 的核心概念之一。理解它有助于你：
> 1. 理解 React 的渲染流程
> 2. 理解为什么某些操作会导致不必要的重新渲染
> 3. 更好地进行性能优化
> 记住：虚拟 DOM 的真正优势在于"最小化更新"，而不是"比直接操作 DOM 快"

### 5.2 React 的 Diff 算法是如何工作的？

**问题**：请详细解释 React 的 Diff 算法的工作原理。

**答案**：

#### Diff 算法概述

React 的 Diff 算法用于比较新旧虚拟 DOM 树，找出最小更新操作。算法基于两个假设：

1. **不同类型的元素产生不同的树**
2. **开发者可以通过 key 暗示哪些元素稳定**

#### 对比规则

1. **不同类型的元素**
   - 卸载旧树，创建新树

```jsx
// 旧
<div>Hello</div>
// 新
<span>Hello</span>
// 结果：完全替换，卸载 div，创建 span
```

2. **相同类型的元素**
   - 只更新属性，不重新创建子元素

```jsx
// 旧
<div className="old">Content</div>
// 新
<div className="new">Content</div>
// 结果：只更新 className
```

3. **列表对比**
   - 使用 key 进行匹配

```jsx
// 旧
<ul>
  <li key="a">1</li>
  <li key="b">2</li>
</ul>

// 新
<ul>
  <li key="b">2</li>
  <li key="a">1</li>
</ul>
// 结果：通过 key 匹配，移动位置，而不是重新创建
```

#### Key 的重要性

```jsx
// 正确使用 key
function TodoList({ items }) {
  return (
    <ul>
      {items.map(item => (
        <li key={item.id}>{item.text}</li>
      ))}
    </ul>
  );
}

// 错误使用 key（使用索引）
function TodoList({ items }) {
  return (
    <ul>
      {items.map((item, index) => (
        <li key={index}>{item.text}</li> // 错误：index 不稳定
      ))}
    </ul>
  );
}
```

#### Diff 算法的复杂度

- **传统算法**：O(n³) 复杂度
- **React 优化后**：O(n) 复杂度

React 通过以下策略实现 O(n)：

1. **只对比同级元素**
   - 不跨层级移动

2. **不同类型不同处理**
   - 类型不同直接替换子树

3. **使用 Key 优化列表**
   - 最小化列表重排

#### Fiber 架构（React 16+）

React 16 引入 Fiber 架构，Diff 算法可以分片执行：

```jsx
// Fiber 的工作流程
function workLoop() {
  while (nextUnitOfWork) {
    nextUnitOfWork = performUnitOfWork(nextUnitOfWork);
  }

  if (pendingCommit) {
    commitAllWork(pendingCommit);
  }
}
```

#### 教学提示

> 理解 Diff 算法能帮助你：
> 1. 理解为什么 key 如此重要
> 2. 避免常见的性能问题
> 3. 更好地组织组件结构
> 记住两个黄金法则：
> - **同级比较**：不要将列表项放在不同的 div 中
> - **稳定 Key**：使用唯一 ID 而不是索引作为 key

## 6. 事件处理

### 6.1 React 事件处理与原生 DOM 事件的区别？

**问题**：React 的事件处理机制与原生 DOM 事件有什么区别？

**答案**：

#### 基本区别

```jsx
// React 事件（使用 camelCase）
<button onClick={handleClick}>Click</button>

// 原生 DOM 事件（使用 lowercase）
<button onclick="handleClick()">Click</button>
```

#### 主要区别

| 特性 | React 事件 | 原生 DOM 事件 |
|------|------------|----------------|
| 语法 | camelCase | lowercase |
| 阻止默认 | return false | preventDefault() |
| 事件委托 | 自动 | 需要手动实现 |
| 事件对象 | SyntheticEvent | 原生 Event |

#### React 事件的优势

1. **事件委托**
   - React 在根节点监听所有事件
   - 不需要在每个元素上绑定事件

```jsx
// React 内部实现（简化）
// 不是在每个 button 上绑定事件
// 而是在 root 容器上绑定一个事件
root.addEventListener('click', (event) => {
  // 根据 target 找到对应的处理函数
  handleEvent(event);
});
```

2. **跨浏览器兼容**
   - React 提供统一的 SyntheticEvent
   - 不需要考虑浏览器兼容性问题

3. **内存优化**
   - 事件处理器在组件卸载时自动清理

#### 事件对象（SyntheticEvent）

```jsx
function HandleEvent(event) {
  // React 事件对象

  console.log(event.currentTarget); // 绑定事件的元素

  // 阻止默认行为
  event.preventDefault();

  // 阻止冒泡
  event.stopPropagation();

  // 检查事件类型
  console.log(event instanceof MouseEvent); // true
}
```

#### key 的作用

1. **帮助 React 识别变化的元素**
2. **提高 Diff 算法效率**
3. **保持组件状态**

```jsx
// 没有 key 的问题

    { id: 1, text: 'Learn React' },
    { id: 2, text: 'Learn JavaScript' }
  ]);

          <li key={index}>
            <input type="text" defaultValue={todo.text} />
          </li>
        ))}
      </ul>
    </div>

// 正确：使用唯一 id
function TodoList() {

    { id: 2, text: 'Learn JavaScript' }
  ]);

    <ul>
      {todos.map(todo => (
        // 正确：使用 id 作为 key
        <li key={todo.id}>
          <TodoItem todo={todo} />
        </li>
      ))}
    </ul>
  );
}
```

#### 复杂列表渲染

```jsx

```

#### 使用索引作为 key 的场景

在某些情况下可以使用索引：

1. **列表是静态的，不会重新排序**
2. **列表项没有唯一标识**
3. **列表顺序不会改变**

```jsx

```

#### 教学提示

> 列表渲染的黄金法则：
> 1. **始终使用 key**，优先使用唯一 ID
> 2. **不要使用索引作为 key**（除非列表是静态的）
> 3. key 必须在兄弟元素中唯一
> 4. key 不会传递给子组件（如果需要，使用其他 prop）

### 9.2 Context 的使用场景和注意事项？

**问题**：React Context 的适用场景是什么？使用时应注意什么问题？

**答案**：

#### Context 适用场景

```jsx
// 1. 主题切换

function App() {
  return (

    </AuthContext.Provider>

// 3. 语言/国际化

```

#### Context 的问题

1. **不必要的重新渲染**

```jsx
// 问题：所有使用 ThemeContext 的组件都会重新渲染
const ThemeContext = React.createContext();

function App() {
  const [theme, setTheme] = useState('light');

  return (
    // 每次 theme 变化，所有消费者都会重新渲染
    <ThemeContext.Provider value={{ theme, setTheme }}>
      <Toolbar />
      <Content />
    </ThemeContext.Provider>
  );
}

// 解决方案：分离 Context
const ThemeContext = React.createContext();
const SetThemeContext = React.createContext();

// 只在 theme 变化时重新渲染
function App() {
  const [theme, setTheme] = useState('light');

  return (
    <ThemeContext.Provider value={theme}>
      <SetThemeContext.Provider value={setTheme}>
        <Toolbar />
      </SetThemeContext.Provider>
    </ThemeContext.Provider>
  );
}
```

2. **闭包陷阱**

```jsx
// 问题：Context 值是对象时，每次渲染都是新对象
function App() {
  const [count, setCount] = useState(0);

  // 每次渲染都会创建新对象
  const value = { count, setCount };

  return (
    <MyContext.Provider value={value}>
      <Child />
    </MyContext.Provider>
  );
}

// 解决方案：使用 useMemo 缓存
function App() {
  const [count, setCount] = useState(0);

  const value = useMemo(() => ({ count, setCount }), [count]);

  return (
    <MyContext.Provider value={value}>
      <Child />
    </MyContext.Provider>
  );
}
```

#### 最佳实践

1. **按功能分离 Context**

```jsx
// 分离为多个小 Context
const AuthContext = React.createContext();
const ThemeContext = React.createContext();
const LocaleContext = React.createContext();
```

2. **使用 useContextSelector（React Labs）**

```jsx
// 选择性订阅，避免不必要渲染
import { useContextSelector } from 'use-context-selector';

function Profile() {
  // 只订阅 user 变化
  const user = useContextSelector(AuthContext, state => state.user);
}
```

#### 教学提示

> Context 是 React 内置的依赖注入机制：
> 1. 适用于"全局"性质的数据（主题、语言、用户）
> 2. 不要用于频繁变化的状态（考虑使用状态管理库）
> 3. 注意分离 Context 避免不必要的重新渲染
> 4. Context 的 value 应该使用 useMemo 缓存

## 10. 性能优化

### 10.1 React 性能优化的方法有哪些？

**问题**：请介绍 React 中常见的性能优化技巧。

**答案**：

#### 性能优化策略

1. **使用 React.memo 缓存组件**

```jsx
// 普通组件
function Button({ onClick, children }) {
  console.log('Button rendered');
  return <button onClick={onClick}>{children}</button>;
}

// 使用 memo 缓存
const Button = React.memo(function Button({ onClick, children }) {
  console.log('Button rendered');
  return <button onClick={onClick}>{children}</button>;
});

// 自定义比较函数
const Button = React.memo(
  function Button({ onClick, children }) {
    return <button onClick={onClick}>{children}</button>;
  },
  (prevProps, nextProps) => {
    // 返回 true 表示 props 相等，不重新渲染
    return prevProps.onClick === nextProps.onClick;
  }
);
```

2. **使用 useMemo 和 useCallback**

```jsx

    onItemClick(id);

        </li>

}
```

```jsx
  const deferredQuery = useDeferredValue(query);
  const isStale = query !== deferredQuery;

  return (
    <div style={{ opacity: isStale ? 0.5 : 1 }}>
      <input onChange={handleChange} />
      <Results query={deferredQuery} />
    </div>
  );
}
```

## 12. Hooks 最佳实践

### 12.1 自定义 Hook 的使用？

**问题**：如何创建自定义 Hook？自定义 Hook 有什么作用？

**答案**：

#### 自定义 Hook 基础

自定义 Hook 是一个使用 use 开头的函数，内部可以调用其他 Hook：

```jsx
function useCustomHook() {
  const [state, setState] = useState(initialValue);

  // 业务逻辑

  return [state, setState];
}
```

#### 常见自定义 Hook 示例

1. **useLocalStorage**

```jsx
function useLocalStorage(key, initialValue) {
  const [storedValue, setStoredValue] = useState(() => {
    try {
      const item = window.localStorage.getItem(key);
      return item ? JSON.parse(item) : initialValue;
    } catch (error) {
      console.error(error);
      return initialValue;
    }
  });

  const setValue = (value) => {
    try {
      const valueToStore = value instanceof Function ? value(storedValue) : value;
      setStoredValue(valueToStore);
      window.localStorage.setItem(key, JSON.stringify(valueToStore));
    } catch (error) {
      console.error(error);
    }
  };

  return [storedValue, setValue];
}

// 使用
function App() {
  const [name, setName] = useLocalStorage('name', '');

  return <input value={name} onChange={e => setName(e.target.value)} />;
}
```

2. **useDebounce**

```jsx
function useDebounce(value, delay) {
  const [debouncedValue, setDebouncedValue] = useState(value);

  useEffect(() => {
    const handler = setTimeout(() => {
      setDebouncedValue(value);
    }, delay);

    return () => clearTimeout(handler);
  }, [value, delay]);

  return debouncedValue;
}

// 使用
function Search() {
  const [query, setQuery] = useState('');
  const debouncedQuery = useDebounce(query, 500);

  useEffect(() => {
    // 使用 debouncedQuery 搜索
    searchAPI(debouncedQuery);
  }, [debouncedQuery]);

  return <input value={query} onChange={e => setQuery(e.target.value)} />;
}
```

3. **useFetch**

```jsx
function useFetch(url, options = {}) {
  const [data, setData] = useState(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);

  useEffect(() => {
    const controller = new AbortController();

    setLoading(true);
    fetch(url, { ...options, signal: controller.signal })
      .then(res => res.json())
      .then(data => {
        setData(data);
        setLoading(false);
      })
      .catch(err => {
        if (err.name !== 'AbortError') {
          setError(err);
          setLoading(false);
        }
      });

    return () => controller.abort();
  }, [url]);

  return { data, loading, error };
}

// 使用
function UserProfile({ userId }) {
  const { data, loading, error } = useFetch(`/api/users/${userId}`);

  if (loading) return <Spinner />;
  if (error) return <Error error={error} />;
  return <div>{data.name}</div>;
}
```

4. **useOnClickOutside**

```jsx
function useOnClickOutside(ref, handler) {
  useEffect(() => {
    const listener = (event) => {
      if (!ref.current || ref.current.contains(event.target)) {
        return;
      }
      handler(event);
    };

    document.addEventListener('mousedown', listener);
    document.addEventListener('touchstart', listener);

    return () => {
      document.removeEventListener('mousedown', listener);
      document.removeEventListener('touchstart', listener);
    };
  }, [ref, handler]);
}

// 使用
function Dropdown() {
  const ref = useRef(null);
  const [isOpen, setIsOpen] = useState(false);

  useOnClickOutside(ref, () => setIsOpen(false));

  return (
    <div ref={ref}>
      <button onClick={() => setIsOpen(!isOpen)}>Toggle</button>
      {isOpen && <div>Content</div>}
    </div>
  );
}
```

#### 自定义 Hook 的规则

1. **以 use 开头**
   - 必须以 "use" 前缀命名
   - 否则 React 无法检查 Hook 调用规则

2. **只在顶层调用**
   - 不要在循环、条件、嵌套函数中调用
   - 确保 Hook 在每次渲染时以相同顺序调用

3. **只从 React 函数中调用**
   - 不能从普通 JavaScript 函数中调用

#### 教学提示

> 自定义 Hook 是 React 中非常重要的代码复用模式：
> 1. 自定义 Hook 让我们能够将组件逻辑提取为可复用的函数
> 2. 常见场景：数据获取、事件监听、本地存储、表单处理
> 3. 一个好的自定义 Hook 应该是纯粹的、易于测试的
> 4. 可以使用 custom hooks 库避免重复造轮子（react-use、ahooks）

## 14. 服务端渲染与Next.js

### 14.1 服务端渲染（SSR）与客户端渲染（CSR）的区别？

**问题**：请解释服务端渲染和客户端渲染的区别，以及它们的优缺点。

**答案**：

#### 渲染方式对比

| 特性 | 客户端渲染 (CSR) | 服务端渲染 (SSR) |
|------|------------------|------------------|
| 首屏加载 | 较慢（需下载 JS） | 较快（直接返回 HTML） |
| SEO | 较差 | 友好 |
| 交互性 | 需要 JS 加载完成后 | 需要 JS hydration |
| 服务器负载 | 较低 | 较高 |
| 开发复杂度 | 较低 | 较高 |

#### 客户端渲染流程

```
浏览器 → 请求 → 服务器返回空 HTML + JS → 浏览器下载 JS → 执行 JS → 渲染页面
```

#### 服务端渲染流程

```
浏览器 → 请求 → 服务器执行 React 组件 → 生成 HTML → 返回完整 HTML → 浏览器显示
```

#### Next.js 基础

```jsx
// pages/index.js (SSR)
export async function getServerSideProps() {
  const data = await fetchData();

  return {
    props: {
      data
    }
  };
}

export default function Home({ data }) {
  return <div>{data.title}</div>;
}

// pages/about.js (Static)
export async function getStaticProps() {
  const data = await fetchData();

  return {
    props: {
      data
    },
    revalidate: 60 // ISR
  };
}

// API 路由
export default function handler(req, res) {
  res.status(200).json({ name: 'John' });
}
```

#### Next.js App Router (Next.js 13+)

```jsx
// app/page.tsx
async function getData() {
  const res = await fetch('https://api.example.com/data');
  return res.json();
}

export default async function Page() {
  const data = await getData();

}

// app/layout.tsx
export default function RootLayout({ children }) {
  return (

    </html>
  );

// app/page.tsx (Client Component)

```

#### 教学提示

> SSR 和 CSR 有各自的适用场景：
> 1. 需要 SEO 的页面使用 SSR（博客、电商详情页）
> 2. 内部工具、仪表盘使用 CSR 即可
> 3. Next.js 提供了完整的 SSR/SSG/ISR 解决方案
> 4. 注意 hydration 不匹配的问题
