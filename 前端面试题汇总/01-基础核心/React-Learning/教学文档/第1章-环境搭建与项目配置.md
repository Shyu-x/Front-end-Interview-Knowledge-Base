## 1.1 Node.js 25.x 安装与配置

### 什么是 Node.js？

Node.js 是一个基于 Chrome V8 引擎的 JavaScript 运行时，让 JavaScript 可以在服务端运行。React 开发需要使用 Node.js 和 npm（Node Package Manager）来管理项目依赖。

### Node.js 25.x 新特性（2025年10月发布）

- **V8 14.1 引擎**：JSON.stringify() 操作性能大幅提升
- **Uint8Array Base64/Hex 转换**：内置支持，无需额外库
- **WebAssembly 优化**：JSPI 支持，提升交互性能
- **可移植编译缓存**：提升大型项目的编译速度

### 安装步骤

#### Windows 系统

1. **下载 Node.js**
   - 访问官网：https://nodejs.org/
   - 推荐下载 **LTS（长期支持）版本** 25.x
   - 当前推荐版本：**Node.js 25.4.0 LTS**

2. **安装 Node.js**
   - 双击下载的 `.msi` 安装包
   - 保持默认设置，点击 "Next" 完成安装

3. **验证安装**
   ```bash
   # 打开命令行（cmd 或 PowerShell）
   node -v    # 显示 Node.js 版本，如 v25.4.0
   npm -v     # 显示 npm 版本，如 10.5.0
   ```

#### macOS 系统

```bash
# 使用 Homebrew 安装
brew install node

# 或下载 macOS 安装包
# https://nodejs.org/
```

#### Linux 系统

```bash
# Ubuntu/Debian
curl -fsSL https://deb.nodesource.com/setup_25.x | sudo -E bash -
sudo apt-get install -y nodejs

# 或使用 nvm 管理多版本
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.0/install.sh | bash
nvm install 25
nvm use 25
```

### npm 基础命令

```bash
# 查看版本
npm -v

# 初始化项目（创建 package.json）
npm init

# 安装包
npm install <package-name>          # 安装到项目
npm install -g <package-name>       # 全局安装
npm install <package-name> --save   # 保存到 dependencies
npm install <package-name> --save-dev  # 保存到 devDependencies

# 更新包
npm update <package-name>

# 删除包
npm uninstall <package-name>

# 查看已安装的包
npm list

# 运行脚本
npm run <script-name>
```

## 1.2 pnpm 包管理器

### 什么是 pnpm？

pnpm 是一个高性能的 Node.js 包管理器，相比 npm 和 yarn：
- **更快**：安装速度比 npm 快 2-3 倍
- **更节省磁盘空间**：使用硬链接共享依赖
- **更严格**：更好的 monorepo 支持

### 安装 pnpm

```bash
# 使用 npm 全局安装
npm install -g pnpm

# 验证安装
pnpm --version  # 显示 pnpm 版本，如 9.x
```

### pnpm 常用命令

```bash
# 安装依赖
pnpm install

# 添加依赖
pnpm add <package>           # 生产依赖
pnpm add -D <package>        # 开发依赖

# 运行脚本
pnpm dev      # 开发服务器
pnpm build    # 生产构建
pnpm preview  # 预览

# 其他命令
pnpm update   # 更新依赖
pnpm remove   # 移除依赖
```

## 1.3 创建 React 项目（使用 Vite 6）

### 为什么要使用 Vite？

Vite 是新一代前端构建工具，2025年最新版本为 **Vite 6.5.x**：

| 特性 | 说明 |
|------|------|
| **开发启动更快** | 基于 ES 模块，无需打包 |
| **热更新更快** | 采用 HMR（热模块替换） |
| **生产构建更优** | 使用 Rollup 进行优化打包 |
| **Environment API** | 更好的环境配置支持 |

### 创建项目步骤

#### 方法一：使用 pnpm（推荐）

```bash
# 创建项目（项目名不能包含大写字母）
pnpm create vite@latest my-react-app -- --template react

# 进入项目目录
cd my-react-app

# 安装依赖
pnpm install

# 启动开发服务器
pnpm dev
```

#### 方法二：使用 npm

```bash
# 创建项目
npm create vite@latest my-react-app -- --template react

# 进入项目目录
cd my-react-app

# 安装依赖
npm install

# 启动开发服务器
npm run dev
```

#### 方法三：使用 yarn

```bash
# 创建项目
yarn create vite my-react-app --template react

# 进入目录
cd my-react-app

# 安装依赖
yarn

# 启动开发服务器
yarn dev
```

### 创建过程中的选项说明

```
? Project name: ...              # 项目名称
? Select a framework: ...        # 选择框架（选 React）
? Select a variant: ...          # 选择变体
  > JavaScript                  # JavaScript 版本
    TypeScript                  # TypeScript 版本（带类型检查）
    JavaScript + SWC            # 使用 SWC 加速
    TypeScript + SWC            # TypeScript + SWC
```

│   ├── index.css          # 全局样式

├── package.json          # 项目配置
├── package-lock.json     # 依赖锁定文件
├── vite.config.js        # Vite 配置文件

### 核心文件详解

#### package.json

```json
{
  "name": "my-react-app",
  "private": true,
  "version": "0.0.0",
  "type": "module",
  "scripts": {
    "dev": "vite",           # 开发服务器
    "build": "vite build",   # 生产构建
    "preview": "vite preview" # 预览构建结果
  },
  "dependencies": {
    "react": "^19.0.0",      # React 核心
    "react-dom": "^19.0.0"   # React DOM
  },
  "devDependencies": {
    "@types/react": "^19.0.0",
    "@types/react-dom": "^19.0.0",
    "@vitejs/plugin-react": "^4.0.0",
    "vite": "^6.0.0"
  }
}
```

#### main.jsx（入口文件）

```jsx
import { StrictMode } from 'react'
import { createRoot } from 'react-dom/client'
import './index.css'
import App from './App.jsx'

// 创建根节点并渲染应用
createRoot(document.getElementById('root')).render(
  <StrictMode>
    <App />
  </StrictMode>,
)
```

#### App.jsx（主组件）

```jsx
import { useState } from 'react'
import './App.css'

function App() {
  const [count, setCount] = useState(0)

  return (
    <div className="App">
      <h1>Hello React 19!</h1>
      <div className="card">
        <button onClick={() => setCount((count) => count + 1)}>
          count is: {count}
        </button>
      </div>
    </div>
  )
}

export default App
```

## 1.8 Vite 6 配置详解

### vite.config.js 基本配置

```javascript
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'

// https://vitejs.dev/config/
export default defineConfig({
  plugins: [react()],

  // 服务器配置
  server: {
    port: 3000,
    open: true,  // 启动时自动打开浏览器
    proxy: {
      '/api': {
        target: 'http://localhost:8000',
        changeOrigin: true,
        rewrite: (path) => path.replace(/^\/api/, '')
      }
    }
  },

  // 构建配置
  build: {
    outDir: 'dist',
    sourcemap: false,
    rollupOptions: {
      output: {
        manualChunks: {
          'react-vendor': ['react', 'react-dom'],
          'antd-vendor': ['antd']
        }
      }
    }
  },

  // CSS 配置
  css: {
    preprocessorOptions: {
      less: {
        javascriptEnabled: true
      }
    }
  },

  // 环境变量
  envPrefix: ['VITE_', 'REACT_']
})
```

### 环境变量配置

创建环境变量文件：

```
#.env                 # 默认环境变量
#.env.local          # 本地环境变量（不提交到 Git）
#.env.development    # 开发环境变量
#.env.production     # 生产环境变量
```

使用环境变量：

```javascript
// 访问环境变量
console.log(import.meta.env.VITE_APP_TITLE)

// 条件判断
if (import.meta.env.DEV) {
  console.log('开发模式')
}

if (import.meta.env.PROD) {
  console.log('生产模式')
}
```

## 1.9 常见问题与解决方案

### 问题1：pnpm install 失败

**解决方案**：
```bash
# 清除缓存
pnpm store prune

# 删除 node_modules 和 pnpm-lock.yaml
rm -rf node_modules pnpm-lock.yaml

# 重新安装
pnpm install
```

### 问题2：端口被占用

**解决方案**：
```bash
# 查找占用端口的进程
netstat -ano | findstr 5173

# 杀掉进程（替换为实际 PID）
taskkill /PID <PID> /F

# 或使用其他端口
pnpm dev -- --port 3001
```

### 问题3：React 版本问题

**解决方案**：
```bash
# 查看当前 React 版本
pnpm list react

# 升级到最新版本
pnpm add react@latest react-dom@latest

# 指定版本
pnpm add react@19 react-dom@19
```

### 问题4：Vite 配置代理

在 `vite.config.js` 中配置：

```javascript
export default defineConfig({

    proxy: {
      '/api': {
        target: 'http://localhost:8000',
        changeOrigin: true,
        rewrite: (path) => path.replace(/^\/api/, '')

```
