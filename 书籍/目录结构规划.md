# 目录结构规划文档（修订版）

## 一、当前问题

1. **目录分散**：存在多个目录包含相似内容
2. **重复内容**：`docs/` 和 `前端面试题汇总/` 内容有重叠
3. **命名混乱**：文件命名格式不统一

## 二、规划原则

- **不删除任何文件** - 只做合并和重新组织
- **保留双目录** - `docs/` 作为精简索引，`前端面试题汇总/` 作为完整内容
- **软链接或引用** - 避免重复内容

## 三、执行方案

### 方案：目录合并策略

由于不能删除内容，采用以下方式：

1. **docs/ 保留但更新内容**
   - 将 `docs/` 改为"精简索引版"
   - 每个文件添加"查看完整版"链接指向 `前端面试题汇总/`

2. **前端面试题汇总/ 保持不变**
   - 保留所有详细内容

3. **添加索引文件**
   - 在 `前端面试题汇总/` 根目录添加 `目录索引.md`
   - 列出所有文件及简要说明

### 需要创建的新文件

| 文件 | 内容 |
|------|------|
| `docs/目录索引.md` | docs 目录的索引 |
| `前端面试题汇总/目录索引.md` | 详细内容的索引 |

### 实际不执行任何删除操作

- 不删除任何现有文件
- 不移动任何文件位置
- 只是在需要时添加索引文件
