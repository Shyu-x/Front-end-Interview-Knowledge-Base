# 规范化流程

> 本章节主要讲解前端工程化中的规范化流程，包括版本控制、代码规范、自动化测试、CI/CD等内容。

## 3.1 Git工作流

### 3.1.1 Git基础

**初始化和配置**
```bash
# 初始化仓库
git init

# 配置用户信息
git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"

# 配置别名
git config --global alias.st status
git config --global alias.co checkout
git config --global alias.br branch
git config --global alias.cm commit
```

**基本操作**
```bash
# 克隆仓库
git clone https://github.com/user/repo.git

# 添加文件
git add filename.txt
git add .  # 添加所有文件

# 提交
git commit -m "Commit message"

# 查看状态
git status

# 查看差异
git diff

# 查看历史
git log
git log --oneline

# 撤销操作
git checkout -- filename  # 放弃工作区修改
git restore filename  # 同上（新版）
git reset HEAD~1  # 撤销最后一次提交
```

### 3.1.2 分支操作

```bash
# 查看分支
git branch

# 创建分支
git branch feature-login
git checkout -b feature-login  # 创建并切换

# 切换分支
git checkout feature-login

# 合并分支
git checkout main
git merge feature-login

# 变基
git checkout feature-login
git rebase main

# 删除分支
git branch -d feature-login
git push origin --delete feature-login  # 删除远程分支
```

### 3.1.3 远程仓库操作

```bash
# 添加远程仓库
git remote add origin https://github.com/user/repo.git

# 拉取
git fetch origin
git pull origin main

# 推送
git push origin main
git push -u origin feature-login
```

### 3.1.4 标签管理

```bash
# 创建标签
git tag v1.0.0
git tag -a v1.0.0 -m "Release v1.0.0"

# 查看标签
git tag
git tag -l "v1.*"

# 推送标签
git push origin v1.0.0
git push origin --tags
```

### 3.1.5 工作流模式

**Git Flow**
```
main:     ────●──────────●──────────●──────────
               ↑          ↑          ↑
            release    release    release
                      /    \
feature:  ●───●───●───●      ↓
                       develop
develop:  ────●───●───●───●───●───●───●───●
```

**GitHub Flow**
```bash
# 从main创建分支
git checkout -b feature/add-login

# 开发并提交
git add .
git commit -m "Add login feature"

# 推送并创建PR
git push -u origin feature/add-login
# 在GitHub创建Pull Request
```

## 3.2 分支管理策略

### 3.2.1 常用分支类型

```
main (或 master)    - 生产环境代码
develop             - 开发主分支
feature/*           - 功能分支
release/*           - 发布分支
hotfix/*            - 热修复分支
bugfix/*            - Bug修复分支
```

### 3.2.2 分支命名规范

```bash
# 功能分支
feature/user-login
feature/add-payment

# Bug修复
bugfix/fix-login-error
bugfix/memory-leak

# 热修复
hotfix/security-patch

# 发布
release/1.0.0
```

## 3.3 代码审查

### 3.3.1 代码审查流程

**Pull Request创建规范**
```markdown
## 描述
简要描述这个PR解决的问题或添加的功能

## 变更类型
- [ ] Bug修复
- [ ] 新功能
- [ ] 破坏性变更
- [ ] 文档更新

## 测试
- [ ] 单元测试通过
- [ ] 集成测试通过
- [ ] 手动测试
```

**代码审查检查点**

1. **代码质量**
   - 代码是否清晰易读
   - 命名是否有意义
   - 函数是否保持单一职责
   - 是否有重复代码

2. **功能正确性**
   - 代码是否实现了预期功能
   - 边界情况是否处理
   - 错误处理是否完善

3. **性能**
   - 是否有性能问题
   - 是否有内存泄漏风险

4. **安全性**
   - 是否有安全漏洞
   - 敏感数据是否正确处理

5. **测试**
   - 是否有足够的测试覆盖
   - 测试用例是否正确

## 4.2 代码规范

### 4.2.1 命名规范

**变量和函数命名**
```javascript
// 变量命名
const userName = 'John';  // 名词，表示数据
const isActive = true;    // 布尔值，以is/has/can/should开头
const userList = [];      // 复数形式表示数组

// 函数命名 - 使用动词或动词短语
function getUserList() {}
function fetchUserInfo() {}
function validateForm() {}
function handleSubmit() {}

// 类命名 - 使用PascalCase
class UserService {}
class ApiClient {}

// 常量命名 - 使用大写和下划线
const MAX_RETRY_COUNT = 3;
const API_BASE_URL = 'https://api.example.com';
```

### 4.2.2 代码组织规范

**文件组织**
```javascript

// 标准库导入
import fs from 'fs';
```

## 4.3 自动化测试

### 4.3.1 测试策略

**测试金字塔**
```
        /\
       /  \
      / E2E \        <- 少量：端到端测试
     /--------\
    / 集成测试 \    <- 中等：集成测试
   /------------\
  /  单元测试    \ <- 大量：单元测试
 /----------------\
```

**测试覆盖率目标**
- 语句覆盖率：80%+
- 分支覆盖率：75%+
- 函数覆盖率：80%+
- 行覆盖率：80%+

### 4.3.2 单元测试示例

**Jest配置详解**
```javascript

module.exports = {
```

## 5.1 CI/CD

### 5.1.1 CI/CD基础概念

**持续集成（CI）**
- 频繁地将代码集成到主分支
- 每次提交触发自动化构建和测试
- 快速发现集成错误

**持续交付（CD）**
- 自动将代码部署到测试环境
- 准备就绪后可以手动部署到生产
- 确保代码始终可部署

**持续部署**
- 自动将代码部署到生产环境
- 完全自动化部署流程

### 5.1.2 GitHub Actions配置

**基础工作流配置**
```yaml
# .github/workflows/ci.yml
name: CI Pipeline

on:
  push:
    branches: [main, develop]
  pull_request:
    branches: [main]

jobs:
  lint:
    name: Lint
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '18'

      - name: Install dependencies
        run: npm ci

      - name: Run ESLint
        run: npm run lint

  test:
    name: Test
    runs-on: ubuntu-latest
    needs: lint
    steps:
      - uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '18'

      - name: Run tests
        run: npm test
```

### 5.2 持续部署

#### 5.2.1 部署到Vercel

```yaml
# .github/workflows/vercel.yml
name: Deploy to Vercel

on:
  push:
    branches: [main]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Install Vercel CLI
        run: npm install -g vercel@latest

      - name: Pull Vercel Environment Info
        run: vercel pull --yes --environment=production

      - name: Build Project
        run: vercel build --prod

      - name: Deploy to Vercel
        run: vercel deploy --prebuilt --prod
```

#### 5.2.2 部署到Netlify

```yaml
# .github/workflows/netlify.yml
name: Deploy to Netlify

on:
  push:
    branches: [main]

jobs:
  build-and-deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '18'

      - name: Install dependencies
        run: npm ci

      - name: Build
        run: npm run build

      - name: Deploy to Netlify
        uses: nwtgck/actions-netlify@v2.0
        with:
          publish-dir: './dist'
          production-branch: main
```

### 5.3 自动化流程

#### 5.3.1 自动版本发布

```yaml
# .github/workflows/release.yml
name: Release

on:
  push:
    tags:
      - 'v*'

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '18'

      - name: Install dependencies
        run: npm ci

      - name: Run tests
        run: npm test

      - name: Build
        run: npm run build

      - name: Publish to npm
        run: npm publish
        env:
          NODE_AUTH_TOKEN: ${{ secrets.NPM_TOKEN }}
```

#### 5.3.2 自动依赖更新

```yaml
# .github/workflows/dependabot.yml
name: Dependabot

on:
  schedule:
    - cron: '0 0 * * 0'  # 每周日检查
  pull_request:

jobs:
  dependabot:
    runs-on: ubuntu-latest
    if: ${{ github.actor == 'dependabot[bot]' }}
    steps:
      - name: Checkout code
        uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '18'

      - name: Update dependencies
        run: |
          npm install
          ncu -u
```
